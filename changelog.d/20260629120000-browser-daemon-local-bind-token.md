- Bind the browser daemon to `127.0.0.1` by default so its powerful commands (`executeScript`, `addCookie`, ...) are only reachable by local processes; override with `--host` only when remote access is intended. Add an optional shared command token via `--token` or the `SYSTEM_TEST_BROWSER_TOKEN` environment variable — when set, browser commands without the matching token are rejected, and the token is never stored in the registry or printed. Commands stay backwards-compatible when no token is configured.
