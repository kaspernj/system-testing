- Add `systemTest.step(name, callback)` (on `Browser`, inherited by `SystemTest`) to group commands into named, optionally nested steps. The callback runs unchanged; on failure the original error is preserved and annotated with the active step path (message `(in step: ...)` plus an `error.systemTestStep` property), and the failure screenshot artifacts record the step. Step boundaries are exposed via `getStepHistory()` / `currentStepPath()` for trace/report tooling.
